W H I T E C A T

The Internet of Things for Humans

What is it?

WhiteCat is both an open hardware platform and a blocks-based graphical programming language.

It is white as in an empty canvas, ready for you to build whatever you can imagine.

What's new about it?

No compiling, real-time automagic deployment

The WhiteCat board can be programmed in a live, dynamic, blocks-based language. Not just interfaced, but really programmed. It works just as you expect it to.

WhiteCat programs are automagically deployed in real time to the board, meaning you can just unplug it at anytime. No compilation, no file transferring. Just unplug the board and whatever was running in the blocks environment will keep running in the board.

Built to be connected

The whole system was designed from the beginning to be able to be deployed in a distributed environment. Boards can communicate with each other -or with other devices- by means of standard protocols, such as the CANbus industry standard or the MQTT network messaging protocol.

These are all built-in in the language, and simple block abstractions let you weave a network of devices in the simplest possible way.

Standard compliant

The WhiteCat is the first PIC-based board that reimplements the whole system stack. Instead of the usual proprietary vendor-locked libraries, we have decided to stand by our principles and base our system in the standard, free -as in freedom and as in gratis- GCC libs.

Everything in the system strives to adhere to standards, which is why we include hardware and software abstractions for the CANbus, MQTT and REST protocols, allowing the board to be seamlessly integrated in any other environment and communicate with any standard-compliant device.

You can build it yourself

The WhiteCat board is so much open hardware that you can actually build it yourself at home! We not only provide the factory-grade schematics, but also provide plans and a detailed process description for you to etch the circuits and solder the board by yourself.

As a single-faced board, the home-brewed version is larger than the factory made one, but it packs pretty much the same capabilities.

Have you ever imagined being able to hand-solder a 32 bit, 200 MHz computer that you can hold in your hand and is powerful enough to even run LiteBSD 4.4?

Why develop a whole new system?

There are several static blocks systems for different hardware solutions that just translate blocks to text. While these help overcome initial barriers to learning programming, they are not live, dynamic environments and the languages they translate to are very low level and lack fundamental computationally rich features, like first class heterogeneous lists, higher order functions or an event system.

To overcome this gap, the Edutec team built Snap4Arduino, which is based on the Snap! programming language and thus inherits all features that we believe an educational, powerful language should have, but on the other hand it lacks the possibility of making the board autonomous.

The WhiteCat blocks environment is conceived to address both needs: having a dynamic, real-time, high-level, blocks-based visual language and, at the same time, allowing you to build an autonomous hardware system while removing the need to compile or translate anything. With the WhiteCat system, the lines between prototype and final product become blurred. The environment lets you build your solutions in an experimental, real-time, iterative way without the need of any additional deployment process.

How does it work?

The WhiteCat is targeted both at professionals, hobbyists and students, and is fitted for educational, industrial or domestic use indistinctly.

To achieve such wide range of applications and user bases, it packs a versatile programming stack meant to fit all profiles.

Blocks on top

We believe programming should be for everyone, and not just for those who hold engineering or computer science degrees, which is why we have developed a blocks-based, dynamic environment based on a stripped-down version of Snap!.

This visual programming environment maps directly to the board, which means all processes happen in the hardware, and the board will keep its state even after unplugging it from the computer and plugging it to a different power source. As in any other dynamic blocks-based language, you can run as many scripts in parallel as you want, and click on any block to run it while other processes are going on.

Since the language running in the board supports computationally rich features, the WhiteCat blocks environment allows you to program in an elegant and way, making use of heterogeneous lists, dictionaries[*], first class blocks[*], and higher order functions.

Lua at the bottom

Lua is a widely used dynamic language that fits like a glove in the WhiteCat environment. Lua is small, fast, robust, embeddable and, most important, it maps beautifully to the dynamic, real-time block-based languages we love, like Scratch and Snap!. Block scripts become coroutines, rings become anonymous functions, lists become tables.

Of course, if you are a Lua programmer, you can just plug yourself into the interpreter and start coding right away. The WhiteCat board includes the latest 5.3 version and you can find the library specifications in our wiki.

What's in the board?

Hardware:

  • PIC32MZ2048EFM064 processor at 200 MHz/330 DMIPS, with 512 Kb of SRAM memory and 2 Mb of flash program memory
  • Float Point Unit and Crypto Engine with a RNG for data encryption/decryption and authentication (AES, 3DES, SHA, MD5, and HMAC)
  • 16 Mb flash memory for file access
  • MicroSD slot for large file access
  • 10/100 Mbps Ethernet port
  • 1 SPI port, 3 UART ports, 1 CAN bus
  • Up to 16 ADC inputs, 12-bit resolution, 18 MSPS
  • Up to 17 I/O ports
  • Up to 6 PWM ports
  • 1 USB console port

Software:

  • FreeRTOS micro-kernel (Real Time Operating System)
  • Lua 5.3.1
  • Lua modules for accessing all hardware peripherals: I/O, ADC, CAN, TIMERS, PWM, SPI, UART...
  • Lua socket module (TCP and UDP)
  • Lua MQTT client module

Who's behind it?

WhiteCat is being developed by a team of engineers, educators and living lab designers, working together to shape all the details comprising hardware specs, low-level software layers, high-level programming language and end-user use cases.

The fact that the WhiteCat team covers multiple fields and disciplines allows us to solve issues in a very dynamic and agile way. This has made building the whole stack from scratch possible.

Citilab - Edutec

The Edutec team from the Citilab has a long history in modifying blocks-based languages. They have developed, among others, S4A, Snap4Arduino and BeetleBlocks.

Their knowledge of Snap!'s codebase has been instrumental in allowing them to build the WhiteCat blocks-based programming environment, which is based on a stripped-down version of this dynamic, visual language.

Iberoxarxa

Iberoxarxa was founded in 2009 with the objective of helping organizations enhance their internal processes by means of free software solutions.

They have been responsible for all the low-level software layers in the WhiteCat system, ranging from the adaptation of the RTOS operating system for the PIC family, up to the Lua interpreter and the implementation of its different modules and libraries.

CSSIberica

CSSIberica has been designing and manufacturing instruments for flight simulators since 2005 under the Simmeters brand, among other services.

Their expertise in digital electronics and, in particular, the PIC ecosystem, has made it possible for them to design and produce the WhiteCat board and all its incremental prototypes.

[*] This system is currently in a pre-alpha development state, and some of the features described here are not yet available.
